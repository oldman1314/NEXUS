# 请求页面视觉优化方案

## 一、问题诊断

### 1.1 元素尺寸比例失调及布局排列问题

| 问题 | 现状 | 对比参考（Data Table / Remote Tools） |
|------|------|------|
| **URL 工具栏过于紧凑** | `padding: 12px 16px`，所有元素挤在一行 | DT 工具栏分多行布局，每行职责清晰 |
| **方法选择器与 URL 输入框视觉割裂** | 两个独立圆角框并排，中间 8px gap | DT 命令栏采用一体化胶囊设计，视觉统一 |
| **Send 按钮缺乏存在感** | 80px 宽的普通按钮，无特殊视觉处理 | DT 发射按钮 30px 圆形 + 发光 + 弹性缩放 |
| **工具按钮（Templates/Import/CodeGen）排列混乱** | 3 个 40px 方形按钮紧贴 Send，无分组 | RT 工具栏左右分区，功能按钮有明确分组 |
| **标签页区域过于朴素** | 纯文字 + 简单下划线，无视觉层次 | DT 分段选择器有滑动高亮块 + 弹性动画 |
| **KV 编辑器行项间距过小** | `gap: 8px`，行高紧凑 | DT 卡片行有左侧指示条 + hover 渐变 + 交错入场 |
| **响应面板头部信息密度低** | 状态码/时间/大小水平排列，视觉权重相同 | DT 面包屑有能量脉冲 + 粒子流动 + 呼吸动画 |
| **空状态设计过于简单** | 一个纸飞机 SVG + 两行文字 | RT 空状态有虚线卡片 + 上浮入场 + 呼吸动画 |
| **分割手柄过于简陋** | 2 个 3px 灰色圆点 | RT 有 6px 拖拽手柄 + accent 色高亮 |

### 1.2 创意性与趣味性不足

| 维度 | 当前状态 | Data Table / Remote Tools 参考 |
|------|---------|------|
| **动态交互** | 仅有 fadeIn/slideUp 基础动画 | DT 有 30+ 关键帧动画（能量流、粒子、3D 翻转） |
| **视觉层次** | 平面化，无深度感 | DT 有 Neumorphism + 光晕效果 |
| **特色图形** | 无 | DT 有能量脉冲线、粒子流、信号点 |
| **状态反馈** | 简单颜色变化 | DT 有发射波/完成波/能量流多阶段动画 |
| **品牌识别** | 无独特视觉语言 | DT "Cyber-Data" 风格，RT 终端主题系统 |

## 二、优化方案

### 方案 A：URL 工具栏重构 — "Signal Bar" 信号栏

**设计理念**：参考 DT 命令栏的胶囊设计，将方法选择器 + URL 输入 + Send 按钮整合为一个一体化信号栏。

**具体改动**：

1. **一体化胶囊容器**
   - 外层容器：`border-radius: 12px`，`border: 1px solid var(--border-color)`
   - 内部无分隔，方法选择器与 URL 共享同一背景
   - 方法选择器区域左侧有 3px 色带指示（保留现有 method-color-band）
   - Send 按钮右侧嵌入，圆形 36px，accent 背景发光

2. **方法选择器融入 URL 栏**
   - 移除独立边框，改为左侧色块 + 文字
   - 右侧用 1px `var(--border-light)` 分隔线与 URL 区域分开
   - 下拉箭头缩小至 12px

3. **Send 按钮圆形化**
   - 改为 36px 圆形按钮，仅显示 Send 图标（无文字）
   - ready 状态：accent 背景 + 发光阴影
   - hover：`scale(1.08)` + 更强发光
   - active：`scale(0.85)` 弹性回弹
   - sending 状态：脉冲发光动画

4. **工具按钮分组**
   - 移至 URL 栏右侧独立区域，用 `1px border-light` 分隔
   - 按钮缩小为 32px，更紧凑
   - 整体视觉权重降低，不抢 URL 栏焦点

5. **聚焦态增强**
   - URL 栏聚焦时：accent 边框 + 3px glow 光晕
   - 空闲时：微弱边框脉冲动画（3s 周期）

**影响文件**：
- `src/components/views/RequestView.tsx` — JSX 结构调整
- `src/components/views/request-view.css` — 样式重构
- `src/components/views/method-select.css` — 方法选择器融入 URL 栏

---

### 方案 B：标签页系统升级 — "Sliding Door" 滑动门

**设计理念**：参考 DT 分段选择器的滑动高亮块设计，替换当前简单的下划线指示器。

**具体改动**：

1. **滑动高亮块**
   - 背景轨道：`border-radius: 8px`，`background: var(--bg-tertiary)`，`padding: 2px`
   - 激活标签：accent 浅色背景块，`border-radius: 6px`
   - 高亮块滑动过渡：`transform: translateX()` + 弹性缓动 `cubic-bezier(0.34, 1.56, 0.64, 1)`
   - 移除 `::after` 底部指示条

2. **标签徽章增强**
   - 激活标签的徽章：accent 背景 + 白色文字 + 微弱发光
   - dot 类型徽章：添加呼吸动画 `dotPulse`

3. **Scripts 下拉标签优化**
   - 下拉箭头旋转动画更明显
   - 展开时标签背景变化

**影响文件**：
- `src/components/views/requestView.tsx` — 标签页渲染逻辑
- `src/components/views/request-view.css` — 标签页样式

---

### 方案 C：KV 编辑器行项升级 — "Signal Row" 信号行

**设计理念**：参考 DT 卡片行的左侧指示条 + 交错入场设计。

**具体改动**：

1. **左侧状态指示条**
   - 每行左侧 3px 圆角指示条，默认 `opacity: 0`
   - 启用的行：accent 色，`opacity: 0.5`
   - hover 时：宽度变为 4px，`opacity: 1`，accent 发光
   - 焦点时：accent 色 + 发光

2. **交错入场动画**
   - 行项入场：`dt-rowEmerge` 风格动画
   - 从下方 8px 渐入，按索引延迟 `calc(var(--row-index) * 30ms)`
   - 使用 CSS 自定义属性 `--row-index` 传递索引

3. **添加按钮增强**
   - 虚线边框改为点状边框 `dotted`
   - hover 时左侧出现 accent 色加号图标
   - 入场动画：从下方 4px 渐入

**影响文件**：
- `src/components/tabs/ParamsTab.tsx` — 传递行索引
- `src/components/tabs/HeadersTab.tsx` — 传递行索引
- `src/components/tabs/kv-editor.css` — 行项样式

### 方案 D：响应面板头部升级 — "Status Core" 状态核心

**设计理念**：参考 DT 面包屑的能量脉冲设计，为响应状态添加动态视觉反馈。

**具体改动**：

1. **状态码徽章增强**
   - 尺寸增大：`padding: 3px 10px`，`font-size: var(--text-sm)`
   - 成功状态：左侧 3px 绿色指示条 + 微弱绿色发光
   - 错误状态：左侧 3px 红色指示条 + 微弱红色发光
   - 入场动画：`scaleIn` + 微弱弹性

2. **响应时间性能指示**
   - 添加 40×4px 热力条（参考 DT 持续时间热力条）
   - <100ms 绿色 / 100-500ms 黄色 / 500ms-1s 橙色 / >1s 红色
   - 填充宽度按比例计算

3. **响应元数据分隔**
   - 状态码、时间、大小之间用渐变分隔线连接（参考 DT 能量脉冲线）
   - 分隔线有微弱的脉冲动画

**影响文件**：
- `src/components/views/ResponsePanel.tsx` — JSX 调整
- `src/components/views/request-view.css` — 响应头部样式

---

### 方案 E：空状态升级 — "Signal Beacon" 信号灯塔

**设计理念**：参考 RT 空状态卡片设计，将纸飞机改为更具品牌感的信号灯塔概念。

**具体改动**：

1. **SVG 插图重设计**
   - 保留纸飞机概念但简化线条
   - 添加信号波纹：3 层同心圆，从内到外逐渐扩散 + 淡出
   - 信号波纹动画：`signalRipple` 2s 周期，3 层延迟 0.4s

2. **文案优化**
   - 标题：`Send a request to see the response`
   - 提示：添加快捷键提示 `Press Ctrl+Enter to send`

3. **背景装饰**
   - 添加微弱的点阵背景（利用现有 `--dot-color` 变量）
   - 点阵在 Immersive 模式下更明显

**影响文件**：
- `src/components/views/RequestView.tsx` — 空状态 SVG 和文案
- `src/components/views/request-view.css` — 空状态样式

---

### 方案 F：分割手柄升级 — "Energy Link" 能量链接

**设计理念**：参考 RT 拖拽手柄设计，增加视觉存在感和交互反馈。

**具体改动**：

1. **垂直分割手柄**
   - 宽度从 8px 减为 6px
   - 默认状态：3 个 3px 圆点垂直排列，`var(--border-color)` 色
   - hover：圆点变为 accent 色 + `scale(1.3)` + 微弱发光
   - 拖拽中：手柄背景 accent-light + 2px accent 色竖线

2. **水平分割手柄**
   - 同理，3 个圆点水平排列
   - 交互反馈与垂直版本一致

**影响文件**：
- `src/components/views/request-view.css` — 分割手柄样式

---

### 方案 G：请求发送动画升级 — "Signal Launch" 信号发射

**设计理念**：参考 DT 命令栏的发射波/完成波设计，为请求发送添加多阶段动画。

**具体改动**：

1. **发送阶段动画**
   - 点击 Send：按钮 `scale(0.85)` 弹性回弹
   - 发射波：URL 栏内从左到右扫光（参考 DT `dt_launchWave`）
   - 进度条：现有进度条增加发光尖端（参考 DT `dt_progressTip`）

2. **成功/失败反馈增强**
   - 成功：URL 栏绿色扫光（从右到左，参考 DT `dt_completeWave`）
   - 失败：URL 栏红色闪烁
   - 现有 `url-flash-success/error` 保留但增强动画

3. **Immersive 模式增强**
   - 发送时：请求面板顶部方法色带脉冲发光
   - 成功时：响应面板右上角绿色光晕（现有，保留）
   - 失败时：响应面板右上角红色光晕（现有，保留）

**影响文件**：
- `src/components/views/request-view.css` — 动画关键帧
- `src/components/views/RequestView.tsx` — 动画状态管理

## 五、设计规范约束

1. **所有修改必须兼容三种视觉模式**：经典 / 暗色 / Immersive
2. **所有动画必须尊重 `prefers-reduced-motion`**：禁用装饰性动画
3. **颜色使用 CSS 变量**：不硬编码色值
4. **保持与 Data Table / Remote Tools 的设计语言一致性**：使用相似的动画缓动、发光效果、交互模式
5. **不引入新的 npm 依赖**
6. **性能优先**：动画使用 `transform` / `opacity`，避免触发重排
