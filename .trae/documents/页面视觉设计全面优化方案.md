# 页面视觉设计全面优化方案

## 一、核心问题诊断

### 1.1 视觉层级倒置（最严重）

当前页面的视觉权重分配完全倒置：

| 元素 | 重要性 | 当前视觉权重 | 应有视觉权重 |
|------|--------|-------------|-------------|
| 标签页导航 | ⭐⭐⭐⭐⭐ | `--text-tertiary`（最弱） | `--text-secondary` + 激活态 accent |
| 操作按钮 (Copy/Raw) | ⭐⭐⭐ | `--text-secondary`（中等） | `--text-secondary` + hover accent |
| 状态徽章 | ⭐⭐⭐⭐ | `--text-tertiary` + `--bg-tertiary`（双重弱化） | `--text-secondary` + accent-light 背景 |
| 工具按钮 (Templates/Import) | ⭐⭐ | `opacity: 0.7`（看起来像禁用） | `opacity: 1` + `--text-tertiary` |
| 格式切换按钮 | ⭐⭐ | `--text-tertiary`（最弱） | `--text-secondary` + hover 变化 |

**结果**：用户无法快速定位关键信息，因为最重要的元素反而最不显眼。

### 1.2 暗色模式对比度系统性不足

| 变量 | 当前值 | 对比度 | 问题 |
|------|--------|--------|------|
| `--text-tertiary` on `--bg-primary` | `#98989d` on `#1c1c1e` | 4.1:1 | 小文本不达标 |
| `--border-light` on `--bg-primary` | `#48484a` on `#1c1c1e` | 1.7:1 | 边框几乎不可见 |
| `--bg-hover` | `rgba(255,255,255,0.06)` | - | 悬停几乎不可感知 |
| `--bg-input` vs `--bg-tertiary` | `#3a3a3c` vs `#3a3a3c` | 0:1 | 完全相同，无法区分 |
| `--bg-sidebar` vs `--bg-primary` | `#1c1c1e` vs `#1c1c1e` | 0:1 | 完全相同，无区域区分 |

### 1.3 硬编码间距泛滥

全项目约 **100+ 处**硬编码间距值，设计令牌变量形同虚设。最严重的文件：
- `request-view.css`：约 40 处
- `script-tab.css`：约 20 处
- `sidebar-base.css`：约 12 处

### 1.4 硬编码颜色不跟随主题

- Send 按钮阴影全部使用 `rgba(0, 122, 255, ...)` 硬编码，暗色模式 accent 是 `#0a84ff`
- 代码语法高亮大量硬编码颜色，亮色模式颜色在暗色模式下几乎不可见
- SVG 下拉箭头使用硬编码 `#86868b` 灰色

## 二、优化方案

### 方案 1：修复暗色模式对比度（P0 — 基础保障）

**修改文件**：`src/styles/global.css`

1. **提升暗色模式文字对比度**
   ```
   --text-tertiary: #98989d → #a1a1a6  (对比度 4.5:1 → 5.2:1)
   --text-secondary: #b0b0b5 → #b8b8bd (对比度 7.5:1 → 8.3:1)
   ```

2. **提升暗色模式边框可见度**
   ```
   --border-light: #48484a → #545456  (对比度 1.7:1 → 2.3:1)
   --border-color: #3a3a3c → #4a4a4c  (对比度 1.4:1 → 1.9:1)
   ```

3. **增强暗色模式悬停反馈**
   ```
   --bg-hover: rgba(255,255,255,0.06) → rgba(255,255,255,0.10)
   ```

4. **区分暗色模式输入框与三级背景**
   ```
   --bg-input: #3a3a3c → #3e3e40  (与 --bg-tertiary 产生区分)
   ```

5. **区分暗色模式侧边栏与主内容区**
   ```
   --bg-sidebar: #1c1c1e → #252527  (与 --bg-primary 产生区分)
   ```

6. **增强亮色模式悬停反馈**
   ```
   --bg-hover: rgba(0,0,0,0.04) → rgba(0,0,0,0.06)
   ```

### 方案 2：修复视觉层级倒置（P0 — 核心体验）

**修改文件**：`src/components/views/request-view.css`

1. **标签页文字颜色升级**
   - `.panel-tab`：`--text-tertiary` → `--text-secondary`
   - `.panel-tab:hover`：`--text-secondary` → `--text-primary`
   - `.response-tab`：同上

2. **标签页徽章增强**
   - `.tab-badge`：`--bg-tertiary` → `--bg-secondary`，`--text-tertiary` → `--text-secondary`
   - `.panel-tab.active .tab-badge`：保持 accent-light 背景

3. **操作按钮增强**
   - `.action-btn`：`--text-secondary` 保持，hover 时 accent 色更明显
   - `.format-toggle-btn`：`--text-tertiary` → `--text-secondary`

4. **工具按钮修复**
   - `.url-bar-tools .save-btn`：移除 `opacity: var(--opacity-default)`，改为 `opacity: 1` + `--text-tertiary`
   - hover 时：`--text-secondary` + `bg-hover`

### 方案 3：统一间距系统（P1 — 规范化）

**修改文件**：`src/components/views/request-view.css`, `src/components/tabs/script-tab.css`, `src/components/layout/sidebar/sidebar-base.css`

**替换规则**：

| 硬编码值 | 替换变量 | 适用场景 |
|----------|----------|---------|
| `2px` | `var(--space-2xs)` | 微间距 |
| `3px` | `calc(var(--space-xs) - 1px)` | 极小间距 |
| `4px` | `var(--space-xs)` | 紧凑间距 |
| `6px` | `var(--space-sm) - 2px` | 小间距 |
| `8px` | `var(--space-sm)` | 标准小间距 |
| `10px` | `var(--space-md) - 2px` | 中小间距 |
| `12px` | `var(--space-md)` | 标准中间距 |
| `14px` | `var(--space-lg) - 2px` | 中大间距 |
| `16px` | `var(--space-lg)` | 标准大间距 |
| `20px` | `var(--space-xl)` | 超大间距 |
| `24px` | `var(--space-2xl)` | 特大间距 |

**重点替换区域**（按优先级）：
1. `.request-toolbar` / `.url-bar-signal` — 用户第一眼看到的区域
2. `.panel-tabs` / `.panel-tab` / `.panel-content` — 高频交互区域
3. `.kv-editor` / `.kv-row` — 数据编辑区域
4. `.response-header` / `.response-toolbar` — 结果展示区域

### 方案 4：修复硬编码颜色（P1 — 主题一致性）

**修改文件**：`src/components/views/request-view.css`, `src/components/tabs/script-tab.css`

1. **Send 按钮阴影**
   - `rgba(0, 122, 255, 0.25)` → `color-mix(in srgb, var(--accent) 25%, transparent)`
   - `rgba(0, 122, 255, 0.35)` → `color-mix(in srgb, var(--accent) 35%, transparent)`
   - 同理替换所有 Send/Cancel 按钮的硬编码阴影

2. **代码语法高亮暗色模式修复**
   - 在 `[data-theme="dark"]` 下为 `.token.comment`、`.token.keyword`、`.token.string`、`.token.function`、`.token.number` 提供高对比度颜色
   - 当前暗色覆盖在 L483-L496，但颜色仍然偏暗

3. **SVG 下拉箭头**
   - `stroke='%2386868b'` → `stroke='var(--text-tertiary)'` 或使用 CSS `currentColor`

### 方案 5：增强交互反馈（P2 — 体验提升）

**修改文件**：`src/components/views/request-view.css`, `src/components/tabs/kv-editor.css`

1. **KV 行删除按钮可发现性**
   - 默认 `opacity: 0` → `opacity: 0.3`
   - hover 行时 `opacity: 0.6`
   - hover 按钮时 `opacity: 1`

2. **方法选择器 hover 增强**
   - 添加边框颜色变化 + 微弱背景变化

3. **格式切换按钮增强**
   - hover 时添加背景色变化 + 微弱边框

## 四、设计规范约束

1. 所有颜色必须使用 CSS 变量，不硬编码色值
2. 所有间距必须使用 `--space-*` 变量
3. 暗色模式文字对比度不低于 4.5:1（WCAG AA）
4. 暗色模式边框对比度不低于 2.0:1
5. 悬停状态必须有可感知的视觉变化（至少 10% 亮度差异）
6. 交互元素不得使用 `--text-tertiary` 作为默认颜色
7. 不引入新的 npm 依赖
