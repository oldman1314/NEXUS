# 替换 HologramBar 设计方案

## 背景

用户希望删除当前 `DataTable` 页面标题栏中的 `HologramBar`（全息投影状态指示器），认为其实现"没有什么创意，且与主题不搭"。用户提供了截图，显示的是一个绿色方块矩阵 + "SYNCED" 文字的组合。

当前 `HologramBar` 的问题：
- 全息投影风格（能量环、粒子、矩阵方块、发光文字）与 Postman-like 的 API 测试工具主题不匹配
- "SYNCED" 等状态文字过于抽象，与数据表格的实际业务场景脱节
- 视觉风格偏向科幻/游戏化，与专业工具的定位不符

## 设计方案：DataPulseBar（数据脉搏指示器）

### 核心理念
将状态指示器重新设计为**"数据流"**主题的微动画，呼应 DataTable 的数据查询、加载、展示流程。

### 视觉隐喻
- **数据源** → **传输管道** → **数据终端**
- 用"数据包"在管道中流动的动画来可视化加载过程
- 用终端上的数据图表来反映结果状态

### 状态映射

| 状态 | 视觉表现 | 颜色 |
|------|----------|------|
| `idle`（待机） | 终端屏幕微光呼吸，管道静止 | 灰色 `#8E8E93` |
| `loading`（加载中） | 数据包从源端流向终端，管道流光 | 蓝色 `#007AFF` |
| `complete`（完成） | 终端显示迷你柱状图，数据包到达 | 绿色 `#34C759` |
| `error`（错误） | 管道断裂闪烁，终端显示红色叉号 | 红色 `#FF3B30` |

### 组件结构

```
DataPulseBar
├── DataSource      (左侧：数据库/服务器图标)
├── PulsePipe       (中间：传输管道 + 流动光点)
│   ├── 管道背景条
│   ├── 流动光点 (loading 时动画)
│   └── 进度填充条
└── DataTerminal    (右侧：终端屏幕)
    ├── 屏幕外框
    ├── 迷你图表 (complete 时显示)
    └── 状态指示点
```

### 动画细节

1. **idle**: 终端屏幕 3s 呼吸动画（opacity 0.3 → 0.6），管道无流动
2. **loading**: 
   - 数据包（4px 圆点）从左侧以 1.2s 周期流向右侧
   - 管道背景产生流光渐变移动效果
   - 终端屏幕亮起（opacity 0.8）
3. **complete**: 
   - 数据包到达终端后，终端内显示 3 根迷你柱状图依次升起（0.3s stagger）
   - 终端底部状态点常亮绿色
4. **error**: 
   - 管道中间出现断裂效果（两段分离 + 红色闪烁）
   - 终端显示红色叉号（SVG path 绘制动画）

### 交互
- hover 时整体放大 1.05 倍，光点速度加快
- 点击无特殊行为（保持与 DroneBar/CircuitBar 一致）

### 技术实现

1. 新建 `DataPulseBar.tsx` + `data-pulse-bar.css`
2. 修改 `TitleBar.tsx`，将 `data-table` 视图映射到 `DataPulseBar`
3. 删除 `HologramBar.tsx` + `hologram-bar.css`（或保留备用）

### 风格一致性
- 与 `DroneBar`（无人机）、`CircuitBar`（电路）保持相同的尺寸规范（~200px 宽，标题栏高度内）
- 使用相同的 CSS 变量系统（`var(--accent)`、`var(--success)` 等）
- 保持 `ErrorBoundary` 包裹

## 实施步骤

1. 创建 `DataPulseBar.tsx` 组件
2. 创建 `data-pulse-bar.css` 样式
3. 修改 `TitleBar.tsx` 替换引用
4. 删除旧的 `HologramBar.tsx` 和 `hologram-bar.css`
5. 验证各状态动画正常

## 备选方案

如果用户不喜欢"数据流"方向，还可考虑：
- **显微镜/放大镜主题**：搜索/查询数据的隐喻
- **漏斗主题**：数据筛选的隐喻
- **简洁数字计数器**：直接显示加载条数/总条数
